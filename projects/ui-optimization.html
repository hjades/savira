Game Lobby UI Optimization

Enhancing User Experience Through Data-Driven Design

Business Context

Our game lobby is the primary touchpoint for users to discover and select games. Analytics revealed that users were spending less time in the lobby and engagement with newer games was below target. This directly impacted our key business metrics: game discovery, session time, and monetization opportunities.

This project aligned with our Q2 strategic initiative to improve discovery and session metrics across our entire game portfolio.

User Research Insights

Using heatmap analysis (Hotjar) and user session recordings, we identified key pain points:

  • Users struggled to distinguish between game categories
  • Interactive elements were difficult to locate
  • Our primarily 45+ demographic found text and buttons too small
  • Visual hierarchy was unclear, causing cognitive overload

Solution Strategy

We approached this redesign with three key principles:

  1. Accessibility First: Designing for our older demographic meant prioritizing larger touch targets, improved contrast, and simplified interactions.
  2. Visual Hierarchy: Creating clear distinction between primary actions (game selection) and secondary actions (settings, profile) through size, color, and position.
  3. Incremental Testing: Instead of a complete overhaul, we tested individual elements with 10% of users to validate improvements before full implementation.

I created a prioritization matrix to determine which UI elements would deliver the highest impact relative to development effort, focusing our initial sprints on the highest-value improvements.

Design Process

The redesign went through multiple iterations:

  1. User Journey Mapping: I worked with UX researchers to document the current lobby navigation flow and identify friction points.
  2. Wireframing: Created low-fidelity mockups using Figma to test multiple layout options.
  3. User Testing: Conducted remote testing sessions with 12 users across different age groups using InVision prototypes.
  4. A/B Testing: Deployed three variants to test button size, placement, and color schemes with real users.
  5. Iterative Refinement: Used heat mapping data to further optimize placement of high-value elements.

Implementation Challenges

The project faced several technical and organizational hurdles:

  • Technical Debt: The legacy codebase had inconsistent UI implementations requiring significant refactoring.
  • Cross-Platform Considerations: Ensuring a consistent experience across Android, iOS, and web versions required close collaboration with three different development teams.
  • Performance Concerns: Initial designs increased load times on older devices. I worked with engineering to optimize asset loading and reduce initial render time by 40%.
  • Stakeholder Alignment: Marketing wanted to maintain prominent placement for promotional content, which conflicted with our simplification goals. I conducted a workshop to align on user-centered metrics that satisfied both teams' objectives.

Key Learnings

  • Small UX improvements can drive significant business impact when focused on critical user journeys
  • Cross-functional alignment is crucial - success required shared understanding across design, engineering, and business teams
  • Progressive enhancement is more effective than complete redesigns for maintaining user comfort while improving experience
  • Cultural considerations matter - Indonesian users had different navigation preferences than Taiwanese users
